# Playwright 全方位测试计划

## 目标
使用 Chromium 浏览器对紫微斗数项目进行全方位测试，验证各项功能是否正常工作。

## 测试环境
- **浏览器**: Chromium (使用用户提供的 `E:\TraeFile\chrome-win\chrome.exe`)
- **前端地址**: `http://localhost:3000` (已在 terminal 4 运行)
- **测试范围**: 命盘生成、AI 聊天、模型选择、深色模式等

## 测试步骤

### 步骤 1: 启动 Playwright 并访问首页
- 使用 Playwright 启动 Chromium 浏览器
- 访问 `http://localhost:3000`
- 验证页面是否正常加载
- 截图保存首页状态

### 步骤 2: 测试命盘生成功能
- 填写出生信息表单:
  - 出生日期: 1990-01-01
  - 出生时间: 12:00
  - 性别: 男
  - 日期类型: 阳历
- 点击生成命盘按钮
- 验证命盘是否正确显示
- 截图保存命盘结果

### 步骤 3: 测试 AI 聊天功能
- 点击 AI 分析按钮打开聊天窗口
- 发送测试消息: "分析一下我的命盘"
- 验证 AI 是否正常响应
- 检查是否没有 RAG 相关的参考资料
- 截图保存聊天界面

### 步骤 4: 测试模型选择功能
- 打开模型选择下拉框
- 验证模型列表:
  - qwen3.5-flash
  - glm-4.7
  - kimi-k2.5
- 验证 deepseek-v3.2 已被移除
- 截图保存模型列表

### 步骤 5: 测试深色模式
- 切换深色模式
- 验证命盘颜色是否正确变化
- 验证聊天窗口颜色是否正确变化
- 截图保存深色模式效果

### 步骤 6: 测试响应式布局
- 调整浏览器窗口大小:
  - 桌面端 (1920x1080)
  - 平板端 (768x1024)
  - 移动端 (375x667)
- 验证布局是否正常适配
- 截图保存不同尺寸的页面

### 步骤 7: 检查浏览器控制台
- 打开浏览器开发者工具
- 检查是否有 JavaScript 错误
- 检查网络请求是否正常
- 验证没有 RAG API 调用

### 步骤 8: 生成测试报告
- 汇总所有截图
- 记录测试结果
- 列出发现的问题（如有）

## 预期结果
- ✅ 首页正常加载
- ✅ 命盘生成正确
- ✅ AI 聊天正常工作，无 RAG 内容
- ✅ 模型列表正确，无 deepseek-v3.2
- ✅ 深色模式正常工作
- ✅ 响应式布局正常
- ✅ 控制台无错误

- [ ] Playwright 能正确连接到 Chromium
- [ ] 所有页面元素能正确加载
- [ ] 用户交互能正常响应
- [ ] 截图能正确保存
