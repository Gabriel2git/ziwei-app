# Brainstorming - 设计构思

## 何时使用

当满足以下条件时，**必须**使用此技能：

1. **新功能开发** - 用户要求添加新功能或特性
2. **复杂任务** - 任务复杂，需要先理清思路
3. **用户意图不明确** - 需要通过提问澄清用户真正想要什么

## 典型场景

- 用户说"我想做一个用户登录系统"
- 用户要求"优化一下性能"
- 用户提出"添加一个新的图表组件"
- 任何需要先思考再编码的任务

## 工作流程

### 1. 理解用户意图

通过提问细化用户的需求：

```
"你能详细描述一下这个功能的具体需求吗？"
"这个功能的主要用户场景是什么？"
"有没有参考的实现或者你喜欢的设计？"
```

### 2. 探索替代方案

提出不同的实现思路供用户选择：

```
方案A: 使用现有的库快速实现
方案B: 从零开始定制开发
方案C: 结合两者，部分定制
```

### 3. 分块展示设计

将设计分成小块逐一展示：

**第一块：核心功能**
- 描述主要功能点
- 确认是否正确

**第二块：数据模型**
- 描述数据结构
- 确认是否满足需求

**第三块：用户交互**
- 描述界面和交互
- 确认是否符合预期

### 4. 保存设计文档

将确认的设计保存为文档，记录：
- 需求描述
- 确定的方案
- 关键技术决策
- 待确认的问题

## 使用方法

当用户提出新需求时：

1. **立即停止编码冲动** - 不要直接开始写代码
2. **提问澄清** - 通过Socratic方法理清需求
3. **展示设计** - 分块展示，逐一确认
4. **记录决策** - 保存设计文档

## 最佳实践

### DO

✅ **先理解再动手** - 确保完全理解用户需求
✅ **分块展示** - 将复杂设计拆分成小块
✅ **提供选择** - 展示不同方案的优缺点
✅ **记录决策** - 将确认的设计写成文档

### DON'T

❌ **不要立即编码** - 先思考再动手
❌ **不要假设** - 有疑问立即提问
❌ **不要过度设计** - 先解决核心需求

## 示例

### 示例1：用户要求添加登录功能

```
用户：我想加一个用户登录功能

我：
1. 确认需求：
   - 需要哪些登录方式？（邮箱/手机/第三方）
   - 需要注册功能吗？
   - 是否需要记住密码？

2. 展示方案：
   - 方案A：使用 NextAuth.js（快速但定制有限）
   - 方案B：自建登录系统（灵活但工作量大）
   - 方案C：使用 Supabase Auth（平衡之选）

3. 确认设计后保存文档
```

### 示例2：用户要求优化性能

```
用户：页面加载太慢了，优化一下

我：
1. 确认问题：
   - 哪个页面最慢？
   - 是首次加载还是交互慢？
   - 有具体的性能指标吗？

2. 分析方案：
   - 方案A：代码分割和懒加载
   - 方案B：API 响应优化
   - 方案C：CDN 和缓存

3. 确认后制定实施计划
```

## 触发条件

此技能在以下情况自动触发：

1. 用户提出新功能需求
2. 用户描述一个复杂问题需要解决
3. 用户使用 "/plan" 命令
4. 任务需要多个文件或较大改动

## 与其他技能的配合

- **writing-plans** - 设计确认后，创建详细实施计划
- **test-driven-development** - 实施时采用TDD方法
- **requesting-code-review** - 完成后进行代码审查
