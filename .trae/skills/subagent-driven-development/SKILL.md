# Subagent-Driven Development - 子代理驱动开发

## 何时使用

当满足以下条件时，**必须**使用此技能：

1. **复杂任务** - 任务需要多个文件和较大改动
2. **并行处理** - 有多个独立的任务需要同时处理
3. **效率优化** - 需要提高处理多任务的效率

## 工作原理

```
用户需求 → 拆分为任务 → 分配给子代理 → 两阶段审查 → 合并结果
```

## 两阶段审查

### 第一阶段：规格合规性

检查子代理的输出是否符合计划：

```
检查清单：
- [ ] 是否按照计划要求实现？
- [ ] 是否覆盖了所有需求？
- [ ] 文件路径是否正确？
- [ ] 功能是否按预期工作？
```

### 第二阶段：代码质量

检查代码质量：

```
检查清单：
- [ ] 代码是否清晰易读？
- [ ] 是否有明显的bug？
- [ ] 是否有安全风险？
- [ ] 是否有性能问题？
```

## 工作流程

### 1. 分析计划

理解需要执行的任务列表：
- 任务数量
- 任务依赖关系
- 优先级

### 2. 分配子代理

为每个任务创建子代理：

```typescript
// 为任务1创建子代理
Task({
  description: "实现用户认证",
  query: "在 src/auth 目录下实现用户认证功能...",
  subagent_type: "vibe-architect-coder"
});

// 为任务2创建子代理
Task({
  description: "创建登录页面",
  query: "在 src/app/login 目录下创建登录页面...",
  subagent_type: "vibe-architect-coder"
});
```

### 3. 执行任务

子代理并行或串行执行任务：
- 并行：独立任务可以同时执行
- 串行：有依赖关系的任务按顺序执行

### 4. 两阶段审查

每个子代理完成后进行审查：

**阶段1：规格检查**
- 确认符合计划要求
- 确认覆盖所有需求

**阶段2：质量检查**
- 代码质量
- 潜在问题

### 5. 迭代修复

如果审查发现问题：
- 返回给子代理修复
- 重新审查
- 继续下一个任务

## 使用方法

当有多个任务需要处理时：

1. **分析任务** - 将大任务拆分为小任务
2. **创建子代理** - 使用 Task 工具创建子代理
3. **等待执行** - 等待子代理完成
4. **审查结果** - 执行两阶段审查
5. **处理问题** - 如果有问题，返回修复

## 子代理类型

根据任务选择合适的子代理类型：

- **vibe-architect-coder** - 架构和代码实现
- **search** - 搜索和研究任务
- **code-reviewer** - 代码审查

## 最佳实践

### DO

✅ **任务独立性** - 确保任务之间没有依赖
✅ **清晰指令** - 给子代理完整的上下文和指令
✅ **两阶段审查** - 先规格后质量
✅ **及时反馈** - 发现问题立即返回修复

### DON'T

❌ **不要跳过审查** - 每个任务都要审查
❌ **不要忽视警告** - 质量问题要及时修复
❌ **不要过度并行** - 有依赖的任务要串行

## 任务模板

```markdown
## 任务: [任务名称]

### 描述
[简短描述任务内容]

### 文件
- 主要文件: [路径]
- 相关文件: [路径]

### 要求
1. [具体要求1]
2. [具体要求2]

### 验证步骤
1. [验证步骤1]
2. [验证步骤2]
```

## 触发条件

此技能在以下情况自动触发：

1. 存在详细的实施计划
2. 用户说"开始实施"或"go"
3. 任务需要多个子代理并行处理

## 与其他技能的配合

- **writing-plans** - 提供详细的任务计划
- **test-driven-development** - 子代理使用TDD方法
- **requesting-code-review** - 人工代码审查
- **verification-before-completion** - 最终验证
